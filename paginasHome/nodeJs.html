comandos básicos no terminal

  • mkdir - criar pasta(usa o comando em seguida o nome da pasta e preciona enter)
  • cd - pasta
  • cd funcoes - para entrar na pasta desejada usa-se o codigo cd e o nome da pasta
  • pwd - saber em qual pasta está
  • -v - saber a versão
  • npm - é o geranciador de pacotes do node
  • npm init - configurar arquivo package.json
  • npm init -y - faz o arquivo de forma automatica
  • npm i - significa install (tambem pode ser escrito install) serve para intalar bibliotecas ou dependencias que estão no packad.json

sobre o arquivo package.json:

esse arquivo vai ter a descrição de todas as dependencias que seu projeto precisa. Baixado dependencia dessa forma: npm i --save axios - usamos o --save para gerar uma area que será colocado as dependencias do seu projeto.

no exemplo acima com o axios a versão aparece do lado da seguinte forma: "exios": "^0.27.2" temos um sinal e uma sequencia com tres numeros. O primeiro é o minor, o segundo major e o ultimo o fix

o sinal passado no exemplo foi o circunflexo. Isso quer dizer que a versão da sua dependencia poderar atualizar automaticamente, apenas o mejor até a versão 99, mas não poderar atualizar o minor, no caso chegar na versão 1 do minor. pode alterar o sinal. Usando o ~ só irá atualizar automaticamente a versão do fix e se não usar nenhum sinal, quer dizer que só vai usar a versão informada

a forma de ter dependencias mostrada ate agora fofram de forma global, mas tem como baixar dependencias necessarias apenas no desenvolvimento da aplicação. usando o mesmo exemplo do axios fariamos o seguinte comando no terminal para baixar: npm i --save-dev axios@0.27.2 -E npm e o i já sabemos o que é, o --save-dev serve para informar que só é necessaria termos as bibliotecas ou dependencias no desenvolvimento. Em seguida informamos a dependencia que queremos baixar (no caso o axios). Depois o sinal @ (que serve para informar a versão que será baixada), e por ultimo o -E para informar que quer que instale exatamente a versão passada.

Usar scripts

no exemplo foi adicionado dois scripts com a mesma função, chamar o nodemon que serve para executar o código sem precisar usar atalhos ou botões, apenas salvando o código. o primeiro script nomeamos como start e o segundo como dev. para startar o script start que é um comando padrão do node vamos dar o comando npm start no terminal. Agora para conseguir startar o script nomeado com o código não padrão temos que chamar usando o rum, da seguinte forma: npm rum dev
